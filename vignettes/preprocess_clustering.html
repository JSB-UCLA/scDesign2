Clustering as a preprocessing step of scDesign2

Tianyi Sun

2021-06-18

Introduction

The model fitting and data simulation of scDesign2 is performed for each cell type separately. Therefore, the partitioning of cell types, or cell clustering, is an important preprocessing step of scDesign2. The partitioning of cell types can be performed based on biological knowledge, e.g., marker genes, or using cell clustering algorithms, e.g., SC3 or the Louvain method as implemented in the Seurat package, or a combination of the two.

In this tutorial, we will use a subset of a real dataset to give a proof-of-concept demonstration of how cell clustering can be performed using the Louvain method from Seurat with default parameters, and evaluated using ROGUE scores and mLISI values. Note that in real applications, the demonstrated procedure may not be an optimal way of clustering for every dataset, due to factors such as how to tune the parameters, which clustering algorithm to use, how to incorporate prior biological knowledge, etc. However, we believe that the following procedure will in general give a good enough clustering result for beginner users so that subsequently, the model fitting and data simulation of scDesign2 can be performed. For users with expertise in cell clustering or domain knowledge in biology, they can use their own cell clustering results before using scDesign2 to simulate data.

Preparation

The dataset that we use is still the single cell RNA-seq dataset that profiles the transcriptome of mouse small intestinal epithelium, where its GEO number is GSE92332. To simplify the analysis, we will select six cell types from the original dataset and only analyze this subset of the data.

  if(is.null(colnames(data_mat))){

}

Clustering

To perform cell clustering, we use the Louvain method as implemented in the Seurat R package. In the following code, the function get_cluster_seurat() is based on one of the tutorials of the Seurat R package. Some of the key parameters are the number of “important” genes to include, the number of PCs to include, and the resolution of the clustering. Here, we use the default parameter values as given in the tutorial. In practice, the clustering quality may be improved through tuning of these parameters.

To evaluate the quality of the clustering result, we use the ROGUE score, which is an entropy-based metric for assessing the purity of single cell populations. The ROGUE score can be calculated for each cell cluster, and a higher value indicates better homogeneity of the corresponding cluster. The ROGUE score calculation is wrapped up in the check_cluster_quality() function.

Another metric to evaluate the separation of cell clusters is the mcLISI value, which is proposed by the single-cell data integration method Harmony. However, this value is computed only according to the 2D-visualiztion plot, and therefore is affected by the dimensionality reduction method. More about this will be discussed later in the visualization section.

# function of using ROGUE scores to check cluster qualities -------------------------------
library(ROGUE)
library(tidyverse)

check_cluster_quality <- function(data_mat, cell_type_sel,
                                  platform = c("full-length", "UMI")){
  platform <- match.arg(platform)
  
  data_mat_sel <- get_submat(data_mat, cell_type_sel)
  
  expr <- data_mat_sel
  rogue.res <- rogue(expr, labels = colnames(expr),
                     samples = rep(1, ncol(expr)), platform = platform)
  unlist(rogue.res)
}

# function of using Seurat to cluster -----------------------------------------------------
library(Seurat)

get_cluster_seurat <- function(data_mat, platform = c("full-length", "UMI"),
                               dims = 1:10, res = 0.5){
  platform <- match.arg(platform)
  submat <- data_mat
  
  if(is.null(rownames(submat)))
    rownames(submat) <- 1:nrow(submat)
  count_seurat <- CreateSeuratObject(counts = submat)
  ### normalization
  count_seurat <- NormalizeData(count_seurat)
  ### select highly variable genes
  count_seurat <- FindVariableFeatures(count_seurat, selection.method = "vst", nfeatures = 4000)
  ### scale the data
  count_seurat <- ScaleData(count_seurat)
  ### PCA
  count_seurat <- RunPCA(count_seurat,
                         features = VariableFeatures(object = count_seurat),
                         verbose = F)
  ### clustering
  count_seurat <- FindNeighbors(count_seurat, dims = dims)
  count_seurat <- FindClusters(count_seurat, resolution = res)
  ### results
  cluster_predicted <- as.integer(Idents(count_seurat))
  
  colnames(submat) <- as.character(cluster_predicted)
  rogue_scores <- check_cluster_quality(submat, unique(cluster_predicted), platform)
  list(clustering_result = cluster_predicted, rogue_scores = rogue_scores)
}
# rogue score for real data with cluster label  -------------------------------------------

Model fitting and data simulation

Based on the above clustering result, we can now proceed to use scDesign2 to fit a model to each cell type and then simulate data accordingly. In the following code, the function get_scDesign2_result() wraps up these two steps.

# use scDesign2 to fit model and simulate data with cluster label -------------------------
library(scDesign2)

get_scDesign2_result <- function(count_mat, cell_type_sel){
  submat <- get_submat(count_mat, cell_type_sel)
  n_cell_new <- ncol(submat)
  cell_type_prop <- table(colnames(submat))[cell_type_sel]
  copula_result <- fit_model_scDesign2(count_mat, cell_type_sel, sim_method = 'copula',
                                       ncores = length(cell_type_sel))
  sim_count_copula <- simulate_count_scDesign2(copula_result, n_cell_new, sim_method = 'copula',
                                               cell_type_prop = cell_type_prop)
  list(copula_result = copula_result, sim_count_copula = sim_count_copula)
}

set.seed(1)
colnames(submat) <- clustering_result$clustering_result
sim_count_cluster_label <- get_scDesign2_result(submat, unique(colnames(submat)))
saveRDS(sim_count_cluster_label, file = paste0("sim_count_cluster_label.rds"))

t-SNE visualization

Finally, we visualize the results using t-SNE, a popular dimensionality reduction method. We will visualize the real data and the simulated data separately and compare the patterns, and also combine them together to see whether they can mix well.

For quantitative evaluation, we will compute the mcLISI values for the real data and the simulated data, as well as the miLISI value for the combined data. The LISI, short for Local Inverse Simpson Index, is proposed in the Harmony paper. The mcLISI is short for median cell-type LISI, which indicates the separation of cell types/clusters on the 2D visualization plot. The value is between 1 and 2, and a lower value indicates a better seapration of cell types/clusters. The miLISI is short for median integration LISI, which indicates the mixing of two datasets. It is also between 1 and 2, and a higher value indicates a better mixing of the two datasets.

library(lisi)
get_lisi <- function(x, subset_label){
  emb_dat <- subset(x, subset = panels == subset_label,
                    select = c("x", "y"))
  # str(emb_dat)
  emb_label <- subset(x, subset = panels == subset_label,
                      select = "labels", drop = FALSE)
  # str(emb_label)
  # str(compute_lisi(emb_dat, emb_label, "labels"))
  compute_lisi(emb_dat, emb_label, "labels")
}

get_colScale <- function(labels){
  cbPalette <- c("#999999", "#E69F00", "#56B4E9", "#009E73",
                 "#F0E442", "#0072B2", "#D55E00", "#CC79A7")
  n_cell_type <- length(levels(labels)) - 2
  # print(str(labels))
  if(n_cell_type > 8)
  {
    colourCount <- n_cell_type
    getPalette <- colorRampPalette(brewer.pal(8, "Dark2"))
    cbPalette <- getPalette(colourCount)
  }else{
    cbPalette <- cbPalette[1:n_cell_type]
  }
  cbPalette <- c(cbPalette, '#FF0000', '#000000')
  names(cbPalette) <- levels(labels)
  scale_colour_manual(name = "labels",values = cbPalette)
}

Indeed, the pattern of the simulated data is less similar to the real data than before, and they also do not mix as well, as indicated by the many non-overlapping red points as well as a lower miLISI value.

However, if we look closely at the 2D visualizations of the real data and compare the cluster pattern of Seurat and the original labels, we can see that they can be related to each other. For example, clusters 5 and 10 correspond to the Goblet cell type, cluster 1 correspond to most of the Stem cell type, cluster 9 correspond to the Tuft cell type, etc. This suggests that instead of discarding the original cluster labels completely, a refinement of the clusters of each cell type can be performed to obtain a better result. This refinement procedure is described in the published scDesign2 manuscript in the section “Refinement of scDesign2 training: calibration of cell types by ROGUE scores”.

Summary

In this tutorial, we have shown that using the default parameter setting of the Seurat clustering procedure can do a relatively good job of partitioning the cells. This can be seen from both the t-SNE visualization plots as well as from the ROGUE scores and the mLISI values. Moreover, checking the ROGUE scores of predefined cell types is useful. If they are too low, cell type refinements can be performed.
